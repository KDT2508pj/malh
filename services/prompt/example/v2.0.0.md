# Prompt: Resume Keyword Extraction with Evidence (Structured JSON) - v2.0.0

## Breaking Change Summary
- 루트 구조가 `meta`, `data`로 변경된다.
- 각 필드에 `confidence`, `evidence_quotes`(근거 문구) 를 포함할 수 있다.
- 기존 v1 파서는 그대로 사용하면 실패하므로, 파서/DB 스키마 업데이트가 필요하다.

## Purpose
- 입력된 이력서 텍스트(resume_text)에서 핵심 정보를 구조화(JSON)한다.
- 각 추출 결과에 대해 **근거(evidence)** 와 **신뢰도(confidence)** 를 함께 제공한다.

## Inputs
- resume_text: 이력서에서 추출한 전체 텍스트(UTF-8)

## Rules
1) 추측 금지
- resume_text에 없는 정보는 절대 생성하지 않는다.
- 불명확하면 value는 "" 또는 0/빈 배열로 두고 confidence는 낮게 설정한다.

2) evidence_quotes 규칙
- evidence_quotes는 resume_text에서 **그대로 복사한 짧은 문구**만 넣는다(최대 2개).
- 문구는 가능한 한 짧게(예: 10~40자) 유지한다.

3) confidence 규칙
- 0.0 ~ 1.0 범위의 number로 출력한다.
- resume_text에 명시적으로 존재하면 0.8 이상을 권장한다.
- 추론/해석이 개입되면 0.6 이하로 낮춘다(단, 추측 생성은 금지).

4) 정규화
- skills는 v1과 동일하게 중복 제거 및 표기 통일을 수행한다.
- 단, evidence_quotes는 원문 그대로 유지한다(정규화된 value와 달라도 된다).

5) 출력 제한
- JSON 이외의 텍스트를 절대 출력하지 않는다.
