# Prompt: Resume Keyword Extraction (Structured JSON) - v1.1.0

## Purpose
- 입력된 이력서 텍스트(resume_text)에서 핵심 정보를 구조화(JSON)한다.
- 결과는 DB 저장 및 질문 생성 파이프라인의 입력으로 사용한다.

## Inputs
- resume_text: 이력서에서 추출한 전체 텍스트(UTF-8)

## Output Format (STRICT JSON ONLY)
아래 스키마를 반드시 만족하는 JSON만 출력한다. 설명 문장/마크다운을 출력하지 않는다.

{
  "position": {
    "target_role": "string",
    "level": "string",
    "years": "number"
  },
  "career": [
    {
      "company": "string",
      "role": "string",
      "period": { "start": "YYYY-MM", "end": "YYYY-MM or PRESENT" },
      "highlights": ["string"]
    }
  ],
  "skills": {
    "languages": ["string"],
    "frameworks": ["string"],
    "databases": ["string"],
    "infra": ["string"],
    "tools": ["string"]
  },
  "education": [
    {
      "school": "string",
      "major": "string",
      "degree": "string",
      "period": { "start": "YYYY-MM", "end": "YYYY-MM or PRESENT" }
    }
  ],
  "projects": [
    {
      "name": "string",
      "summary": "string",
      "tech_stack": ["string"],
      "my_role": ["string"],
      "impact": ["string"]
    }
  ],
  "certificates": [
    { "name": "string", "issuer": "string", "date": "YYYY-MM" }
  ]
}

## Extraction Rules (강화)
1) 추측 금지
- resume_text에 없는 정보는 절대 생성하지 않는다.
- 불명확한 값은 "" 또는 []로 둔다(타입은 반드시 유지한다).

2) 정규화 규칙
- skills 항목은 중복 제거 후 표기 통일을 수행한다.
  - 예: "springboot", "Spring Boot" → "Spring Boot"
  - 예: "mysql", "MySQL" → "MySQL"
- 약어는 일반적으로 널리 쓰이는 형태로 통일하되, resume_text에 명시된 표기를 우선한다.

3) 기간 표기
- period.start / period.end는 가능한 경우만 채운다.
- end가 현재 진행이면 "PRESENT"로 표기한다.
- 날짜가 월 단위로 알 수 없으면 ""로 둔다(예: "2023"만 있는 경우).

4) projects 추출 기준
- 프로젝트 이름이 없고 설명만 있을 경우, name은 ""로 두고 summary에 핵심을 담는다.
- tech_stack은 resume_text에 등장한 기술만 나열한다.
- summary는 1문장으로 정리한다(줄바꿈 없음).

5) 출력 제한
- JSON 이외의 텍스트를 절대 출력하지 않는다.
- 값에 줄바꿈을 넣지 않는다(필요하면 문장을 한 줄로 정리한다).

## Quality Checklist (모델 자체 점검)
- 모든 필드가 존재하는가? (없으면 빈 값으로라도 채웠는가)
- years는 number 타입인가?
- skills가 중복 없이 정규화되었는가?
- resume_text에 없는 회사/학교/자격증을 생성하지 않았는가?
